CONCEPT · WHY SKILLS?02 / 12
Before skills

Why do we use agent skills?

Repeated tasks require repeated explanation.

MONDAY

“Summarize these notes in our usual format.”

Long prompt. Format instructions. Reminder to extract owners and deadlines.

NEXT TIME

“Use the same structure as last time.”

The agent may not know which structure. The user repeats corrections.

AGAIN

“Don’t invent missing details.”

The same guardrails are restated. Good workflows stay informal and hard to share.

CONCEPT · DEFINITION03 / 12
What is an agent skill?

A reusable job aid for your AI assistant.

Instead of explaining the same task from scratch, you package the instructions once, improve them over time, and reuse them whenever that task comes up.

TriggerWhen to use the skill
StepsHow to complete the task
RulesConstraints and things to avoid

GETTING STARTED · INSTALLATION07 / 12
How to install a skill

Three common installation paths.

Start with the easiest route your agent supports. Some platforms can install directly from a skill repository, while others expect you to place files in the correct directories or use a package-manager workflow.

01Easiest

Ask the agent to install from a repo

If the agent supports repository installs, you can simply point it to a skill repo and let it handle the setup.

  • Provide the repo URL or repo name
  • Let the agent fetch and install the skill
  • Best when setup is built into the agent UX
02Manual

Copy the skill into the right directory

If direct installs are not supported, place the skill files into the agent’s local or global skill folders yourself.

  • Download or clone the skill package
  • Move it into a specific or global skills directory
  • Useful when you want explicit control
03Ecosystem

Use package managers or custom repos

Some ecosystems provide CLI tools, package managers, or curated registries such as skills.sh to make distribution easier.

  • Install via a package or registry command
  • Browse custom skill repositories
  • Useful for team workflows and repeatable setup
Choose the pathUse the simplest method your agent platform actually supports.
Expect variationThe exact folders, commands, and repo conventions differ across agents.
Then verifyAfter installation, confirm the skill appears in the available skill list or activates successfully.

The key idea

SKILL.md is the contract for the skill: when to activate, how to work, and what good output looks like.

Identity + activation Name the skill, define its purpose, and make the activation cues easy for the agent to recognize.
Inputs + workflow List the context or files the skill expects, then spell out the process and decision logic clearly.
Guardrails + output Constrain behavior, define the target structure, and point to samples or references when useful.
Design principle Keep one skill focused on one kind of job so it remains reusable, legible, and easy to maintain.
Mental model
A skill is more than a prompt.
The file blends metadata, activation guidance, task instructions, and quality constraints into one reusable unit.

Then
Tell it how to perform the task well.

WRITING · DECISION POINT11 / 12
Source, adapt, or create

Source · Adapt · Write

Pick the lowest-effort path that still matches the workflow. Start from an existing skill when possible, adapt when the fit is close, and write when the workflow is specific.

01standard task

Source

Install an existing skill when the task is common and the output format is already well understood.

Recommended sources

Course repoWorkshop-provided starter skills and examples.
Internal libraryApproved team or organization skills.
agentskills.ioStandard docs, examples, and ecosystem links.
skills.shPackage-manager or custom repository workflow.
GitHub reposUseful public examples; inspect before use.
Ask the agent

“Find or install a skill for turning messy meeting notes into structured summaries.”

02close fit

Adapt

Edit SKILL.md when the task is similar but your tone, examples, standards, or output format differ.

Edit the parts that make it yours

TriggerClarify when the skill should activate.
InputsAdd the fields or documents your workflow uses.
ToneSet audience, voice, and wording preferences.
OutputChange sections, tables, naming, or format.
ExamplesAdd one realistic input/output pair.
Ask the agent

“Adapt this SKILL.md for our weekly client update format and ask me only for missing details.”

03unique workflow

Write

Create a new skill when the workflow has specific steps, decision rules, terminology, or quality checks.

Give the AI agent enough raw material

WorkflowWhat repeated task should the skill perform?
Pain pointWhat goes wrong without a skill?
InputsWhat will users provide at runtime?
OutputWhat final format should it produce?
RulesWhat must it do, avoid, or ask about?

“Interview me about this workflow, then draft a focused SKILL.md with trigger, steps, guardrails, and examples.”
